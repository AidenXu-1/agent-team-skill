# 更新日志

本仓库保留最新可安装版本的说明,不维护历史版本清单。

## 当前版本 - 2026-07-02

- 强化多会话协作的读取路由:新会话先运行 `agent_team_read.py onboard`,只读本部门必读文件和人工预写的 YAML 元数据摘要。
- `agent_team_read.py find` 使用精确字段匹配和标签成员匹配,避免 `unblocked` 误命中 `blocked` 这类检索误判。
- 明确报告、审核报告、专项结论、决策记录的统一 frontmatter 字段和只读边界。
- 保留自动 / 人工两种会话模式;自动模式只依赖创建会话、发送短唤醒、设置标题这类基础能力。
- 收敛报告体系:不是所有部门都默认产出正式报告;审核层必须产出审核报告,执行层按复杂度产出工作报告。
- 明确测试不通过后的返工分流:用户体验 OK 后,纯代码 / 质量 / 异常路径问题可由统筹部同步后自主派开发返工;涉及体验取舍、范围、安全、成本、发布等事项仍等用户确认。
- 明确收口与 commit:交接记录不等于 commit;用户确认正式收口后,统筹部检查工作区并按当前节点相关变更 commit 存档。
- 明确设计预览边界:设计部交付的是设计意图可视化,不得声称等同真实 App UI;真实 UI 验收以运行中 App、真实路由、构建或打包态截图为准。
